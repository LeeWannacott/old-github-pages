3. Describe what the DOM is and an example of how you might interact with it.

The HTML DOM outlines the structure of Objects that exist in a browser window (Chrome, firefox, etc...) and web page/document of which you can call methods and properties with. Like for example you might want to get the width and height of the browser window which you can get by using the Window Object's property Window.screen.width ,or/and Window.screen.height.

4. Explain the difference between accessing data from arrays and objects.

Data can be accessed from Objects by using dot notation. Object.property to access properties; which is a variable belonging to an Object and Object.method() for methods; which is a function you can call that belongs to an Object.

Arrays are accesed by using square brackets with an index number. Like so: Array[0]

Example accessing Object's properties with dot noation or brackets:


var vehicle ={ boat : 'Boaty McBoatFace' plane : 'Boeing 747' car : 'Ford Escort MK II' }
vehicle.boat or vehicle['plane'] returns: Boaty McBoatFace and Boeing 747

Example accessing array with indexing:


const cars = ["Saab", "Volvo", "BMW"];
let x = cars[0]; // x = "Saab"

5. Explain what functions are and why they are useful.

functions take inputs and have a series of steps/processes that do something with the input and then return an output (except for pure functions which have no side-effects to the input).

Think of baking a cake which takes ingredients as input (parameters in function) and inside the function it will do operations (like mixing ingredients, putting it in the oven) to bake the cake and then returns an output (a baked cake).



functions are useful because it allows us to encapsulate data inside of the function so that processes dont affect other parts of our code and cause potential bugs.
